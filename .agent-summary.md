# PR #755 remediation — failing `test` CI check

## Root cause

The root `package.json` `test` script (`vitest run`) has no build prerequisite
for the `@aiclarity/shared` workspace package. `vitest.config.ts` has a
`resolve.alias` that redirects `@aiclarity/shared` imports to source
(`packages/shared/src/index.ts`) — but that alias only applies to imports
*vitest itself* transforms. `packages/minspec/tests/migrate-approvals.test.ts`
spawns the real `scripts/migrate-approvals.ts` in a **separate Node process**
(`execFileSync('npx', ['tsx', SCRIPT], ...)`), which resolves `@aiclarity/shared`
through ordinary Node module resolution — i.e. the package's `main` field,
`out/index.js`. On a fresh checkout that file doesn't exist yet (nothing has
run `tsc` for `packages/shared`), so the subprocess throws
`MODULE_NOT_FOUND`. Vitest surfaces this as an unhandled error and the whole
run exits non-zero, even though every individually-counted test still passes.

`npm run build` never showed this because `packages/minspec`'s own `prebuild`
script (`build:shared`) happens to build the shared package as a side effect.
`npm test` has no equivalent step, so a bare `npm ci && npm test` on a fresh
checkout reliably fails.

## Fix

Added a `pretest` script to the root `package.json` that builds
`@aiclarity/shared` before `vitest run` — the same build CI's own `test` job
already runs manually as a separate step (for the coverage provider's
resolution, per its inline comment), now applied automatically to any `npm
test`/`npm run test` invocation, not just the coverage-flavored CI command.

## Verification

- `rm -rf packages/shared/out && npm test` → exit 0 (151/151 test files,
  2969 passed, 2 skipped), repeated twice for flake-check.
- `npm run lint` → clean.
- `npm run build` → clean.
- `npm run validate` → passes (pre-existing non-fatal warnings only, unrelated
  to this change).

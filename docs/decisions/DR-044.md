# DR-044: SealBox — named, and split into its own repo

> **Born `proposed` per DR-029** (acceptance is a separate human act — *MinSpec: Accept
> ADR*). The underlying product decisions are already in motion (sealbox.dev registered,
> repo + site live); this ADR records them for review/acceptance.

## Context

[DR-015](DR-015.md) accepted a **third Tier-1 "Execute" extension** (autonomous
agent dispatch) and — via its OQ-3 — placed it **inside** the public
`AIClarityAU/minspec` monorepo as `packages/agent-execute`, shipped in the Pro pack,
sharing `@aiclarity/shared`. Its name was deferred (OQ-4, [#66](https://github.com/AIClarityAU/minspec/issues/66)),
with `aiclarity.agent-execute` / "AgentSystem" as working placeholders. Its
load-bearing security substrate is specified in [SPEC-019](../../specs/agent-execute/SPEC-019-execution-substrate/requirements.md)
(credential-free sandbox, host-side broker, attestation) and the reality-check
reviewer in [SPEC-016](../../specs/agent-execute/SPEC-016-reality-check/requirements.md).

Two things changed on 2026-06-28:

1. **The product was named.** A #66 analysis (SEO + competitive + security-wedge scan,
   16 candidates, adversarial vetting) found the open lane is *"an agent you can trust
   because it physically can't betray you"* (credential boundary). The founder chose
   **SealBox** — an on-pattern coined compound (seal+box, scanning beside Min+Spec /
   Scrooge+LLM) — and **registered `sealbox.dev`** (Cloudflare). Marketplace id is
   clear; scoped npm `@aiclarity/sealbox` is free.
2. **The founder elected to give SealBox its own repo**, exactly as
   [DR-027](DR-027.md) did for ScroogeLLM — rather than keep it in the monorepo.

This DR records both, and supersedes the relevant DR-015 open questions.

## Decision

1. **Name: SealBox** (brand mixed-case; lowercase `sealbox` for repo/dir/ids).
   Planned ext id **`aiclarity.sealbox`** (retires working `aiclarity.agent-execute`).
   Domain **sealbox.dev**. This **resolves DR-015 OQ-4 / #66** (name no longer deferred).
2. **SealBox lives in its own PRIVATE repo `AIClarityAU/sealbox`** (`~/code/sealbox`),
   mirroring [DR-027](DR-027.md). This **supersedes DR-015 OQ-3** (which placed it at
   `packages/agent-execute` in the monorepo). Private-until-launch because the
   credential-boundary design (SPEC-019) is the pre-launch differentiator — same
   rationale DR-027 used for the proxy moat.
3. **The MinSpec monorepo stays PUBLIC.** The Pro pack references `aiclarity.sealbox`
   **by marketplace id** (location-agnostic), as it already does for `aiclarity.scroogellm`.
4. **DR-015's thesis is otherwise intact** — three-verb line (Specify → Optimize →
   Execute), tier-gated HITL differentiator, single dispatch owner. Only the *home*
   of the Execute extension changes (monorepo → own repo).
5. **The sealbox.dev marketing + waitlist site ships now**, in the new repo (CF Pages
   project `sealbox-dev`, live; waitlist → Workers KV). It does not wait on the code
   migration.

## What has and has not moved (evidence discipline)

- **Done:** the `AIClarityAU/sealbox` repo + `sealbox.dev` site + waitlist are live and
  verified. Name + domain committed.
- **NOT done (tracked):** the agent-execute **specs/DRs/research** (`specs/agent-execute/`
  SPEC-016 + SPEC-019, `EPIC-007`, the agent-execute-specific research under
  `docs/research/`) still live in the **public** monorepo. Until migrated, SPEC-019's
  security design remains public (as ScroogeLLM's was for ~1 week before DR-027's scrub).
  Migration + history-scrub is a follow-up, **not** part of this DR.

## Alternatives considered

- **Keep it in the monorepo (DR-015 status quo).** Simplest; preserves OQ-3. Rejected:
  the founder wants SealBox as its own product with its own identity + a private
  pre-launch surface, matching the per-product repo pattern (MinSpec public, ScroogeLLM
  private, SealBox private).
- **Public sealbox repo.** Rejected for now: the credential-boundary substrate (SPEC-019)
  is the moat-shaped differentiator; keep private until launch, like ScroogeLLM. Flip to
  public at/with launch if desired.
- **New name in the monorepo, no repo split.** Rejected: resolves the name but not the
  founder's "own repo" instruction; leaves the pre-launch design public.

## Consequences

- **Positive:** SealBox gets a clean home + private pre-launch surface; the pack stays
  location-agnostic (marketplace-id reference); MinSpec keeps its public/open posture.
- **Negative / cost:** a **migration** is now owed (specs/EPIC/DRs/research minspec →
  sealbox, with tombstones + a history scrub per the DR-027 playbook). Until then the
  SealBox security design is public. The three bundled `@aiclarity/shared` copies
  concern from DR-015 still applies.

# DR-055: Spec Kit conventional conformity as a migration lever

> **Born / stays `proposed` per DR-029** — acceptance is a separate human act
> (*MinSpec: Accept ADR*). The **constitution amendment in §5 is a proposal to
> redline at Accept-ADR**; it is not applied to `.minspec/constitution.md` until
> this DR is accepted. Nothing about our deliberate divergences (§3) is weakened
> by this DR — it draws the line between *surface* (conform) and *identity*
> (diverge).

## Context

GitHub **Spec Kit** (`github/spec-kit`) is the incumbent spec-driven-development
toolkit. A large share of MinSpec's likely users will have driven a Spec Kit project
first — its `/specify → /plan → /tasks` loop is where the audience learns SDD. Every
place MinSpec's *surface* differs from Spec Kit's for no load-bearing reason is pure
**switching cost**: a file that isn't where they expect, a command that doesn't
autocomplete, a folder named differently.

A conformance audit (2026-07-13, Spec Kit `main` ≈ v0.12.x) found the delta below.
"Identity" = divergence we keep on purpose; "surface" = divergence that is only cost.

| Dimension | Spec Kit | MinSpec (today) | Class |
|---|---|---|---|
| Per-feature artifacts | `spec.md`, `plan.md`, `tasks.md`, **`research.md`**, **`data-model.md`**, **`quickstart.md`**, **`contracts/`** | `requirements.md`, `design.md`, `tasks.md` | **surface** |
| Slash separator | dot — `/speckit.specify` | dash — `/minspec-specify` | surface (minor) |
| Command set | 10 incl. `constitution`, `checklist`, `taskstoissues`, `converge` | 6 (missing those 4) | **surface** |
| Agent-skill form | `speckit-specify` (dash) | `minspec-specify` (dash) | **already conformant** |
| Meta dir | `.specify/` | `.minspec/` | identity |
| Constitution path | `.specify/memory/constitution.md` | `.minspec/constitution.md` | identity |
| Spec id / dir | flat `specs/001-feature/` | nested `specs/<project>/SPEC-004-name/` | identity (DR-027) |
| Command semantics | runs Spec Kit's own flow | shims re-route to MinSpec phases | identity |
| Brand prefix | `speckit` | `minspec` | identity |
| Enforcement | generate-time, agent-driven | **editor/commit/CI-time, deterministic** | identity (G-7, DR-004/014) |

**The tension with G-7.** Goal **G-7** names Spec Kit as the thing we differentiate
*against* ("editor-native SDD … the differentiator vs spec-kit / Kiro"). This DR does
**not** contradict G-7: we differentiate on **enforcement mechanism** (deterministic,
write-time, never-wrong — the moat), and we should look **as familiar as possible** on
the **artifact/command surface** so a Spec Kit user recognises the workspace and reaches
the moat with near-zero relearning. Familiar surface, stronger engine.

## Decision

**Adopt Spec Kit conventional conformity as the default: mirror Spec Kit's artifact
names, folders, and command surface unless a recorded decision says our model requires
otherwise.** Divergence becomes the exception that must cite a reason, not the default.

Concretely, split into three buckets:

### A. Adopt now — additive, non-breaking, endorsed

1. **`data-model.md`** — a recognised (optional) spec artifact: the entities/schema for
   a spec. Sits alongside `design.md`. Maps to Spec Kit's `data-model.md`.
2. **`contracts/`** — a recognised (optional) subfolder inside a spec dir for
   interface/contract payloads (Zod/TS types, OpenAPI, event shapes). Maps to Spec Kit's
   `contracts/`. Directly serves **Contract-Driven Development** ("define contracts
   before implementation" — a global-register methodology, not a local DR) — this is
   conformity *and* methodology win, not just cosmetics.
3. These are **optional** artifacts (present when the spec has a data model / crosses a
   boundary), validated when present, never required by tier alone.

### B. Adopt next — additive command parity

4. **`/minspec-constitution`** — author/update `.minspec/constitution.md` from the
   command surface (we have the file, not the command). Closes a real Phase-1 gap
   (constitution today needs the ext UI or hand-editing).
5. **`/minspec-checklist`** — a checklist phase (Spec Kit `checklist`). Lower priority.
6. `taskstoissues` / `converge` — **evaluate later**; `taskstoissues` overlaps our
   dispatch/triage, `converge` semantics need study. Not adopted here.

### C. Open questions — do NOT decide in this DR (breaking / contentious)

7. **Artifact vocabulary rename** `requirements.md → spec.md`, `design.md → plan.md`.
   High conformity value (these are the *first* files a Spec Kit user looks for) but
   **breaking**: touches every existing spec, validator, tree provider, layout, and
   `spec-kit`-storage-mode code. Needs its own migration DR + spec. **Open.**
8. **Slash separator** dot vs dash (`/minspec.specify` vs `/minspec-specify`). Spec Kit's
   *slash* form is dotted; its *skill* form is dashed (we match the skill form). Marginal;
   dot may collide with tool parsing. **Open — likely keep dash.**
9. `.minspec/` → `.specify/`, `SPEC-NNN` → flat `NNN`, `constitution` under `memory/`:
   **rejected as identity**, not open. `.minspec` and `SPEC-NNN` (DR-027) are load-bearing.

## Keep divergent — identity, explicitly NOT conformed (§3)

Deterministic editor/commit/CI enforcement (G-7), the never-wrong signpost, the
`.minspec/` store, `SPEC-NNN` ids + project nesting (DR-027, cross-project collision),
and the `minspec` brand prefix. Conforming these would erase the product. A Spec Kit user
should feel *"same shapes, but the rules actually fire in my editor"* — that contrast is
the pitch.

## §5 — Proposed constitution amendment (redline at Accept-ADR)

Add one **Principle** (directional, allows the §3 exceptions) — **not** an Invariant.
Rationale: an Invariant is an absolute never-break rule; conformity has deliberate,
recorded exceptions (§3), so encoding it as an Invariant would be self-contradictory. The
existing Principle *"User is always open to a better way — Push-back is welcome"* is the
same register.

Proposed addition to `## Principles`:

> - **Conform to Spec Kit conventions by default.** Mirror Spec Kit's artifact names,
>   folders, and command surface so a Spec-Kit user transitions with near-zero
>   relearning. Diverge only where a recorded decision says our model requires it —
>   deterministic write-time enforcement (G-7), `SPEC-NNN` ids (DR-027), the `.minspec`
>   store, the never-wrong signpost (DR-055 §3). Familiar surface, stronger engine.

## Consequences

- **Positive.** Lower switching cost for the primary inbound audience; `data-model.md` +
  `contracts/` also advance CDD; `/minspec-constitution` closes a Phase-1 command gap.
- **Cost.** Bucket A/B is additive (safe). Bucket C (vocab rename) is a large breaking
  migration if later taken — deliberately deferred.
- **Positioning discipline.** Every future "should we match Spec Kit here?" now has a
  default answer (yes, unless recorded) instead of ad-hoc divergence.
- **Reversible?** The direction is a Principle (cheap to amend). The additive artifacts
  are optional (no forced migration). Low blast radius — hence a DR, not just a spec.

## Clarifications (2026-07-15)

Clarify pass before Accept-ADR. Each is a concrete answerable question with a
resolution or a tracked follow-up; vague concerns stayed in §1–§3.

1. **Does any Bucket C open question (7–9) block Accept-ADR of this DR?**
   **No.** §3 and Bucket C are *explicitly excluded* from this DR's decision —
   it records the direction and adopts A/B only. C being open is by design, so
   acceptance is unblocked. The one remaining act is the human *MinSpec: Accept
   ADR* (which applies the §5 Principle). → not blocking.

2. **Slash separator (Q8) — final lean?** **Keep dash** (`/minspec-specify`).
   The *skill* form is already dashed and conformant (table row), and a dot may
   collide with tool parsing. Recorded as a lean; formal close travels with the
   breaking-migration decision → **#686**. Not a blocker.

3. **Are `data-model.md` / `contracts/` required by any tier?** **No.** Optional,
   validated only when present, never required by tier alone (§A.3). The validator
   must be *symmetric* — accept-when-absent, validate-when-present — to avoid the
   recurring validator-asymmetry defect class. Impl is **#684** (OPEN, not built —
   honest state: recognised in this DR, code pending).

4. **Is Bucket B already delivered?** The two adopted commands exist as agent-skills
   (`minspec-constitution`, `minspec-checklist` present in the skill surface); **#685**
   is `agent-done`. `taskstoissues` / `converge` are **explicitly not adopted** — a
   deferred evaluation, no owner action pending. Verify end-to-end wiring at #685 close
   (skill-exists ≠ command-wired — evidence discipline).

5. **Does adopting A/B need a new spec?** **No.** Additive, low blast radius;
   **#684/#685** carry the implementation. Only Bucket C needs its own DR + spec → **#686**.

6. **§5 — Principle vs Invariant, settled?** **Settled in-DR:** a **Principle**
   (it carries recorded exceptions in §3, so it cannot be an absolute Invariant).
   Applied to `.minspec/constitution.md` at Accept-ADR only, not before.

**Clarify verdict:** zero open items block acceptance. Ready for *MinSpec: Accept ADR*
(human act); Bucket C stays deferred under #686.

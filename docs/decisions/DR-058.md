# DR-058: Autonomous PR remediation in the drain (fix, re-push, re-review — never merge)

## Context

The continuous drain (#239) triages the inbox and dispatches `agent-ready` issues,
each producing a PR via `dispatch-issue.sh`. Nothing then acted on PRs that came
back with a **problem**: an `ai-review:changes` verdict, a red CI check, or a branch
gone stale behind `main`. Those PRs sat until a human hand-fixed each — the exact
backlog the drain exists to prevent, one layer up.

The founder's ask: *the drain should look at and fix PRs with problems like
conflicts, `ai-review:changes`, failed checks.* This is a new **autonomous
capability** — the drain now writes to existing PR branches, not just opens new ones
— so it needs a recording decision (DR-359 ADR filter) and an explicit statement of
where the HITL line stays.

## Decision

**The drain sweeps open, non-draft PRs and auto-remediates *fixable* problems in
place, then re-pushes so CI re-reviews. It never merges, never edits a human PR, and
never auto-resolves a conflict.** Implemented as `scripts/remediate-pr.sh`, invoked
per-PR by the drain's third cycle stage.

### 1. What it fixes (and how)

| Problem | Action |
|---|---|
| `ai-review:changes` (label or red `ai-review` check) | credential-free dev agent addresses the reviewer findings → re-push → CI re-reviews |
| Failing CI checks (test/lint/build/validate) | dev agent reproduces locally (RCDD), fixes the root cause → re-push |
| Behind base | mechanical `git merge origin/main` (no agent) → re-push |

### 2. What it deliberately does NOT do

- **Never merges.** Remediation only *pushes*; the merge keystroke stays human
  (never-wrong / HITL). This is orthogonal to — and does not turn on — the
  deny-by-default **auto-merge** gate (DR-033 §6 / SPEC-024), which remains OFF.
  Remediation *feeds* the review gate; it does not bypass it.
- **Never auto-resolves conflicts.** LLM conflict resolution can silently mismerge.
  A `CONFLICTING`/`DIRTY` PR is surfaced (it already carries `needs-human-review`),
  never touched.
- **Never edits a human PR.** Scope-gated to automation branches (`agent/*`,
  `fix/*`, `feat/*`). A hand-crafted contributor PR is out of bounds.
- **Never mutates `ai-review:*` labels.** CI-as-bot owns them (#600); the re-push
  re-triggers the CI reviewer, which applies the verdict under its own provenance.

### 3. Security model — reused, not re-invented

Identical to `dispatch-issue.sh`: the agent is **credential-free** (no gh / push /
remote / network); the parent does every credentialed op; the **pre-publish egress
guard** (#358) fail-closes on any secret/exfil hit before the push. That guard's
orchestration is the single shared `scripts/lib/agent-egress.sh` (extracted in #747),
so both publish channels — issue dispatch and PR remediation — run the identical
scan and cannot drift (#750). Failing-checks remediation feeds the agent only the
failing check *names* and has it reproduce locally; CI log text (an untrusted-output
injection channel) is never piped into the prompt.

### 4. Bounded autonomy

- **Default-on** (`MINSPEC_DRAIN_REMEDIATE_PRS=1`), off-switch is the same env var
  set to `0`. Consistent with the drain being opt-in per machine already (#239).
- **Attempt cap** — at most `MINSPEC_REMEDIATE_MAX_ATTEMPTS` (default 2) automated
  attempts per PR, counted via marker comments; after that the PR is left for a
  human. No runaway quota, no infinite re-remediation loop.
- **Quota-aware** — a Claude usage-limit signal during remediation pauses the cycle
  with the same backoff as dispatch (#609).
- **Escalation** — a stuck agent escalates once to opus (DR-355), then surfaces to a
  human rather than shipping a partial fix.

## Consequences

**Positive:**
- The reviewer/CI backlog drains without a human touching each PR, while the human
  keeps the only irreversible act (merge). The autonomy is bounded and reversible.
- Realizes the #358 no-drift rationale by giving `agent-egress.sh` its second
  consumer (#750).

**Negative:**
- The drain now writes to PR branches autonomously — a larger blast surface than
  opening new branches. Mitigated by: automation-branch scope, credential-free
  agent + egress guard, attempt cap, and never-merge.
- A remediation agent can push a wrong-but-green fix; the independent CI reviewer is
  the backstop, and the human still merges. Same trust model as issue dispatch.

## Alternatives considered

- **Surface-only (status quo).** Rejected — leaves the backlog the drain exists to
  prevent; the founder ask is explicitly to *fix*, not just flag.
- **Also auto-merge remediated PRs.** Rejected here — merge is the deny-by-default
  gate's job (DR-033 §6 / SPEC-024), which stays OFF and human-gated; this DR does
  not turn it on.
- **Also remediate human PRs / auto-resolve conflicts.** Rejected — human PRs are
  out of scope, and LLM conflict resolution is a mismerge risk; both are surfaced.

## Costly to Refactor

Cheap to reverse — one new script plus a thin third stage in `drain-inbox.sh`, gated
by one env var; undoable in well under a day by setting `MINSPEC_DRAIN_REMEDIATE_PRS=0`
or reverting the stage. No schema, no stored state, no cross-repo contract. Recorded
because it adds an autonomous write path a future maintainer relies on and because it
must state, on the record, where the merge/HITL line stays.

# DR-059: Gate commit-message prose deferrals (blocking) — the surface DR-040 did not cover

## Context

Two mechanisms leak un-tracked work, on **two different surfaces**:

1. **DR-document follow-ups.** A DR's `## Follow-ups (tracked)` bullets that carry no
   issue/spec ref. [DR-040](DR-040.md) governs this: on DR save, un-materialized
   bullets **auto-create** their issues (friction-free), and only genuinely broken
   refs surface. DR-040 deliberately **rejected a blocking gate** here — "only DR-012
   approval blocks in MinSpec; a second blocking gate for bookkeeping" — because the
   author's curated list can be materialized *for* them, so friction is unwarranted.

2. **Commit-message / PR-body prose deferrals.** A commit that says, in prose, *"CI
   files held back for a separate PR"* / *"out of scope, follow-up later"* and cites
   **no** issue. There is no curated list here to auto-file from — the deferral is
   free-text buried in a commit body, and the work it defers becomes invisible the
   moment the PR merges. This is the exact mechanism behind the prose-only-follow-up
   leak (the CI-scope deferral that shipped untracked). **DR-040 never addressed this
   surface** — its auto-create is scoped, by its own Decision §3, to the curated
   `## Follow-ups (tracked)` section, "never over `## Consequences` or prose."

[#725](https://github.com/AIClarityAU/minspec/pull/725) added a `commit-msg` gate for
surface (2). The #725 review (architect) correctly flagged that a *blocking* prose gate
looks like the alternative DR-040 rejected, and that no DR recorded the reversal. This
DR is that record — and it draws the line: **DR-040's "no block" verdict is about
surface (1); it does not bind surface (2), which DR-040 never considered.**

## Decision

**A commit whose message body DEFERS work in prose must cite a tracked follow-up
(`#NNN` / "tracked in …") or explicitly assert none ("Follow-ups: none" / "nothing
deferred"). The `commit-msg` hook blocks otherwise. This is a *second, narrow* blocking
gate — deliberately, and only for the commit-prose surface.**

### 1. Why blocking here, where DR-040 chose non-blocking there

Surface (1) has a curated list the tooling can materialize *for* the author — so
auto-create removes the friction entirely. Surface (2) has **nothing to auto-file
from**: the deferral is unstructured prose whose target work is unknown to the tooling.
The only friction-free options are "do nothing" (the leak) or "ask the human to name
the follow-up." At **commit time** is the sole **actor-agnostic** interception point —
it catches `-m`, editor, heredoc, and agent commits alike (DR-037's reason for owning
gates in the shell tier), which a DR-save hook or a PR-open Action cannot. The escape
is one line, so the gate is friction-*light*, not friction-*heavy*.

### 2. Bounded reach — this does NOT reopen "everything blocks"

The gate fires **only** when deferral language is present **and** no follow-up ref/none
assertion accompanies it. A commit with no deferral prose is never touched. It shares
the `MINSPEC_GATE_OFF=1` bypass with the RCDD gate (DR-003). So the "only DR-012 blocks"
principle is preserved in spirit: routine commits are unblocked; only a commit that
*self-declares* a deferral yet leaves it untracked is stopped.

### 3. False-positive mitigations (the #725 review's second finding)

The gate reads only the **human-authored body**, not the `git commit -v` verbose diff
(everything from the `>8` scissors line down is stripped first) — so diff text that
happens to contain "follow-up" / "out of scope" can never false-trigger. The
satisfied-escape set accepts explicit negations ("nothing deferred", "no follow-ups")
in addition to `#NNN` and "Follow-ups: none", so a legitimate *"out of scope for X but
handled here"* is cleared without a fake issue. The handled-here escape requires an
affirmative verb (`handled|done|fixed|addressed`) before *here / in this PR|commit|change*,
so it cannot swallow the `not in this (pr|commit)` deferral trigger it would otherwise be
a superstring of — a genuine *"not in this PR"* still blocks (#725 review, tested).
Residual breadth (a broad phrase list, escapable by one line) is **intentional**:
a rare "add `Follow-ups: none`" prompt is the accepted cost of never leaking a real
deferral. This mirrors the RCDD gate's own broad-but-escapable posture (DR-003/DR-037).

### 4. Relationship to DR-040 (explicit)

DR-040 stands, unchanged, for surface (1). This DR does **not** revert it; it **scopes**
it: DR-040's "no second blocking gate" applies to DR-document materialization, not to
commit-message prose. Both can hold at once — auto-create for curated DR lists,
blocking-with-cheap-escape for commit prose — because they intercept different leaks at
different points.

## Consequences

**Positive:**
- Closes the commit-prose-leak channel at the only actor-agnostic point, deterministically (no LLM in the decision path).
- Records the reversal the #725 architect asked for, and disambiguates it from DR-040 so a future reader sees the two surfaces are distinct, not contradictory.

**Negative:**
- A second blocking gate exists at commit time. Mitigated: fires only on self-declared deferrals, one-line escape, shared `MINSPEC_GATE_OFF=1` bypass.
- Broad phrase matching can still prompt on an unusual phrasing; accepted per Decision §3, and the escape is cheap.

## Alternatives considered

- **Advisory warning only (no `exit 1`).** Rejected — a warning at commit time is
  noise without the guarantee; the leak this exists to stop already slipped past
  human discipline once. The whole value is that it *blocks*.
- **Scope to a curated commit section (mirror DR-040 §3).** Rejected — commit messages
  have no curated follow-up section, and mandating one is *more* friction than "cite a
  ref or say none." The prose is where the deferral is written; that is what must be
  gated.
- **Do nothing; rely on the local PreToolUse hook.** Rejected — the PreToolUse hook is
  session-local (this harness only); the commit-msg gate is the actor-agnostic,
  teammate-inherited backstop that catches editor/agent commits the hook never sees.

## Costly to Refactor

Cheap to reverse — one shell gate in the scaffolded `commit-msg` hook plus its tests;
undoable in well under a day by deleting the block (the RCDD gate around it is
untouched). No schema, no stored state, no cross-repo contract. Recorded because it
adds a blocking rule a future contributor relies on and because it resolves an
apparent conflict with DR-040 that would otherwise re-surface.

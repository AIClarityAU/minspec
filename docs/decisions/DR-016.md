# DR-016: AI-assisted epic backfill

## Context

DR-013 shipped registered epics, but a project that adopts the feature mid-life
has dozens of existing specs/ADRs/issues carrying no `epic:` reference. Tagging
them by hand is the exact tedium the tool should remove. We want MinSpec to
**propose** an epic taxonomy + an artifact→epic mapping, and to **offer** it
during onboarding.

The obvious tension: good taxonomy-guessing wants semantic reasoning over titles
and prose — an LLM job — but **DR-004 Tier 0** forbids any network/AI in the core
`packages/minspec`. Does this break the air-gapped promise?

**It does not.** DR-004 already resolves it. `claude -p` is **Tier 1** (local tool
delegation), the same category as `gh` (already shelled via `execFile` in
`backlog.ts`) and `claude --bg` (named explicitly in DR-004). The extension makes
**zero outbound connections**; the locally-installed `claude` binary owns its own
networking and auth. No invariant is rewritten and no customer promise is broken,
provided backfill stays Tier 1: opt-in, clearly labeled, and gracefully degrading
when `claude` is absent. The air-gapped core gains no AI dependency.

## Decision

Ship epic backfill as a **two-engine, offer-only, human-gated** feature.

### Two engines

1. **Tier 0 — heuristic pass (always available, air-gapped).** Cluster existing
   specs/ADRs into candidate epics from signals already in the repo:
   - spec-kit subdirectory name (`specs/minspec/epic-grouping/` → epic
     "Epic Grouping"),
   - shared title/tag token overlap (reuse the Jaccard tokenizer behind
     `findSimilarAdrs`),
   - existing `epic:` refs and DR↔spec `Triggered by:` cross-links.
   Pure file-system. No `claude`, no network.

2. **Tier 1 — AI pass (opt-in, needs `claude`).** Shell `claude -p` with a
   compact digest of all artifacts (id, title, kind, path, first prose
   paragraph) and a strict "emit ONLY JSON matching this schema" instruction.
   Parse + validate into the same proposal shape. On any failure (binary absent,
   non-JSON, timeout) **fall back to the heuristic proposal** — never error.

Both engines produce the same `BackfillProposal` contract, so the review +
apply path is engine-agnostic.

### Offer-only onboarding (core stays Tier 0)

`auto-bootstrap` is already a detect-and-offer system. Add a
`hasUnbackfilledEpics()` detector and a bootstrap step that **offers** backfill.
The `claude -p` call runs only on explicit user action — never during first-run
itself, so onboarding remains air-gapped.

### Human-in-the-loop before any write (DR-012 ethos)

Backfill **proposes**; it never silently rewrites frontmatter. The proposal
(candidate epics + per-artifact mapping with rationale/confidence) is surfaced
for review; only on explicit approval does `applyBackfill` create the EPIC-NNN
files, insert `epic:` frontmatter into the mapped artifacts, and regenerate the
epic INDEX. Idempotent; touches only artifacts in the approved proposal; an
artifact that already has an `epic:` is left untouched unless the user opts to
override.

## Consequences

- New module `lib/epic-backfill.ts` (proposal types, heuristic engine, AI engine,
  apply). Tier-1 `claude` detection mirrors `isGhAvailable()`.
- New command `minspec.backfillEpics` + an onboarding offer step.
- A generic `setArtifactEpic(filePath, ref)` frontmatter-insert helper (mirrors
  `setAdrStatus`), reused for specs and ADRs.
- First in-extension `claude -p` delegation (vs. scripts/) — establishes the
  Tier-1 LLM-CLI pattern other features may follow. Guard: availability check +
  graceful fallback are mandatory, never assumed.
- Marketplace copy / docs must label backfill as an optional Tier-1 feature
  ("AI-enhanced; requires Claude Code") and state the heuristic pass works without
  it.

## Alternatives considered

- **AI-only.** Simpler, but the feature is dead without `claude` — no air-gapped
  path. Rejected: a core-adjacent feature should have a Tier-0 mode.
- **Heuristic-only.** Stays purely in Tier 0, but weak taxonomy guesses (no
  semantic grasp of what a spec is *about*). Rejected as the sole engine; kept as
  the fallback + offline mode.
- **Auto-apply the mapping.** Fastest, but an AI silently rewriting frontmatter
  across every doc is exactly the trust failure DR-012 guards against. Rejected:
  HITL review is mandatory.
- **Rewrite the Tier-0 invariant to allow AI in core.** Unnecessary — DR-004's
  tiering already permits `claude -p` as Tier 1. Rejected: would weaken the
  air-gapped promise for no gain.

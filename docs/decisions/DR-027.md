# DR-027: ScroogeLLM is private; MinSpec monorepo stays public

## Context

The `harvest316/minspec` monorepo is **public**. It was framed as a two-extension
monorepo (MinSpec + ScroogeLLM) plus an extension pack. MinSpec is the open, free,
Tier-0 SDD tool — public by design ([DR-004](DR-004.md)). ScroogeLLM is the freemium
proxy whose **defensible IP — the proxy-layer logic and the measurement methodology
behind it — cannot be public.** That extends beyond code: the design specs and
competitive research describe the approach in enough detail to be copied.

The public repo also gave visitors no signal that nothing is shipped yet.

## Decision

1. **MinSpec monorepo stays PUBLIC.** It hosts MinSpec (open), `packages/shared`
   (classifier engine), and the `packages/extension-pack` manifest.
2. **Everything ScroogeLLM moves to a separate PRIVATE repo**
   (`harvest316/scroogellm`): source, the spec (`specs/scroogellm/`), `EPIC-008`,
   `DR-007` (plugin arch), `DR-010` (telemetry), and the scrooge competitive/legal
   research under `docs/research/`. The extension pack references
   `aiclarity.scroogellm` **by marketplace ID**, so it is location-agnostic.
3. **Removed the `packages/scroogellm/` scaffold stub** from the public repo (it was
   harmless — `package.json`/`LICENSE`/`tsconfig`, zero source — but implied the
   source belonged here). Not a build dependency; build unaffected.
4. **Tombstones** replace the moved register entries (`DR-007`, `DR-010`, `EPIC-008`)
   so the sequential register stays gap-free and numeric cross-references resolve.
5. **Public git history was scrubbed** (`git filter-repo`, force-push) to purge the
   moved files' historical blobs. Mitigation, not guarantee: the docs were public
   for ~1 week, so assume some external copy may exist.
6. **Public repo is honestly labelled "under construction"** — README banner +
   shields badges + repo About/topics.

## Alternatives considered

- **Whole monorepo private.** Simplest, zero leak surface — but kills MinSpec's
  open/public positioning and makes a public WIP signpost moot. Rejected: MinSpec is
  meant to be open.
- **Private scrooge as a git submodule inside the public monorepo.** Keeps the slot;
  code stays private. Rejected: leaks existence/name, adds submodule complexity for a
  solo dev, for no gain over a clean separate repo.
- **Leave docs public / redact in place.** Rejected: the moat is described as an
  implementable method in the design + research; partial redaction is error-prone and
  the docs track a private product anyway.

## Consequences

- The "two-extension monorepo" framing is retired; this repo is **MinSpec + shared +
  pack manifest**. README, package table, and license table updated accordingly.
- `DR-007` / `DR-010` / `EPIC-008` survive as tombstones pointing to the private repo;
  prose references in `DR-013` / `DR-015` still resolve.
- The two merged agent branches that carried scrooge content
  (`agent/issue-1`, `agent/issue-21`) were deleted from origin; the orphan `badges`
  branch (no scrooge) was left intact.
- History rewrite changed every commit SHA on `main`. Any other clone must
  `git fetch && git reset --hard origin/main` (or re-clone). A pre-scrub backup bundle
  is retained locally.
- Interacts with [DR-015](DR-015.md) (packaging of the separate Execute extension) and
  [DR-018](DR-018.md) (per-package licensing — the MIT row no longer lists scroogellm).

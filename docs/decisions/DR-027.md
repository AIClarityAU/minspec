# DR-027: ScroogeLLM source is private; MinSpec monorepo stays public

## Context

The `harvest316/minspec` monorepo is **public**. It was framed as a two-extension
monorepo (MinSpec + ScroogeLLM) plus an extension pack. MinSpec is the open, free,
Tier-0 SDD tool — public by design ([DR-004](DR-004.md)). ScroogeLLM is the freemium
proxy whose **moat is the proxy layer itself** — [redacted], where the
measurement is the [redacted], not text compression (see
`project_scrooge_moat_principle`). That code cannot be public.

State at decision time: `packages/scroogellm/` held **only scaffold** —
`package.json`, `LICENSE`, `tsconfig.json`, **zero source**. Git history (3 commits)
touched only those stubs. **Nothing secret had ever been pushed** — this is a
forward-looking structure decision, not leak remediation.

## Decision

1. **MinSpec monorepo stays PUBLIC.** It hosts MinSpec (open), `packages/shared`
   (classifier engine), and the `packages/extension-pack` manifest.
2. **ScroogeLLM source moves to a separate PRIVATE repo** when proxy coding begins.
   The extension pack references `aiclarity.scroogellm` **by marketplace ID**, so it
   is location-agnostic and needs no source in this repo.
3. **Remove the `packages/scroogellm/` stub now** — it is harmless (no code) but its
   presence implies the source belongs here. Removing it makes the public repo's
   structure match the decision. Not a build dependency of any package; build
   unaffected.
4. **Public repo is honestly labelled "under construction"** — README banner +
   shields badges + repo About/topics — so visitors know nothing is published and
   structure churns.

## Alternatives considered

- **Whole monorepo private.** Simplest, zero leak surface — but kills MinSpec's
  open/public positioning and makes a public WIP signpost moot. Rejected: MinSpec is
  meant to be open.
- **Private scrooge as a git submodule inside the public monorepo.** Keeps the slot;
  code stays private. Rejected: leaks existence/name, adds submodule complexity for a
  solo dev, for no gain over a clean separate repo.

## Consequences

- The "two-extension monorepo" framing is retired; this repo is **MinSpec + shared +
  pack manifest**. README, package table, and license table updated accordingly.
- ScroogeLLM's **public spec** (`specs/scroogellm/`) and design docs
  ([DR-008](DR-008.md), [EPIC-008](../epics/EPIC-008-scroogellm.md)) remain public for
  now — but they may themselves leak the moat. Audited separately (see Follow-ups).
- Extension-pack packaging stays guarded until `aiclarity.scroogellm` is published
  (existing guard in `packages/extension-pack/package.json` is unchanged).
- Interacts with [DR-015](DR-015.md) (packaging of the separate Execute extension) and
  [DR-018](DR-018.md) (per-package licensing — the MIT row no longer lists scroogellm).
